01 / Foundation
The Core Idea
What Product+ive is, the gap it fills, and why the Positivity Meter is the product's soul.
One-Line Pitch
Product+ive is an interactive rulebook for your brain — it teaches you the 20 most proven productivity, focus, learning, and study rules, then actively coaches you through each one with timers, trackers, and habit nudges, all feeding into a single Positivity Score that grows the more you show up.

The Problem

Millions of people have read about the Pomodoro Technique, the Feynman Method, or Spaced Repetition — but reading about a rule and actually practising it are completely different things. There is no app that bridges the gap between knowing a productivity rule and doing it with guided assistance, session tracking, and positive reinforcement.

❌ Without Product+ive
You watch a YouTube video about Pomodoro. You try it once, forget the timer, get distracted, and never build the habit. The rule stays in your notes app, unused.
✓ With Product+ive
You open the app, tap Pomodoro, start a 25-min guided session. The app notifies you when to break. Completing it adds to your Positivity Meter. You come back tomorrow because the streak is real.

The Three Pillars

1
Learn — A living library of productivity science
20 rules across 4 categories. Each rule has a description, a how-it-works explanation, and an active "Start Rule" mode. Users browse by category on the home screen visual grid.
2
Do — Guided rule sessions with logic engines
Rules are not just text — they are interactive. Pomodoro launches a countdown timer. Spaced Repetition schedules your reviews. Eat the Frog prompts you to identify your hardest task. Each rule has purpose-built logic, built from reusable engines.
3
Grow — The Positivity Meter rewards every action
Every completed session, every task crossed off, every rule practised earns Positivity Points. The meter fills visually and tracks your momentum. It is the emotional spine of the app — the reason to come back.

Logo Reading — PRO+ive

The logo is a clever typographic puzzle: PRO + a duck-mascot with clock hands + +ive = PRODUCTIVE. The duck body replaces the letters "DUCT" while a clock (productivity = time) overlays it. The + is both the positive symbol and the app's defining character — positivity is built into the name and the icon. This is the app's personality: smart, a bit playful, deeply intentional.

🦆
Mascot Note for UI designer: The duck can appear as a subtle mascot character in onboarding, empty states, and achievement moments (e.g. when the Positivity Meter reaches a milestone). Universal Pain Points

10 / Logic Engine
Reusable Logic Engines
The core of Product+ive's engineering. Instead of building 20 unique rule UIs from scratch, we define 6 reusable engines that all 20 rules map to. Build each engine once — then configure it per rule.
🔧
Engineering Principle: Every rule gets a RuleConfig object defining which engine to use and what parameters to pass. The engine component reads the config — no rule-specific code is needed except the config itself.

Engine 1 — Countdown Timer Engine

⏱
CountdownTimerEngine
For time-boxed focus sessions
Parameters: workDuration, breakDuration, cycles, autoStartBreak, label

UI: Large circular countdown ring + minutes:seconds display + Start/Pause/Skip controls + cycle indicator (e.g. "Round 2 of 4")

Behaviour: Counts down, vibrates on completion, auto-transitions to break if configured, awards points per completed cycle

Notifications: Local notification 60s before end ("Session ending soon"), notification on break start, notification when break ends

Engine 2 — Interval / Reminder Engine

🔔
IntervalReminderEngine
For periodic action reminders
Parameters: intervalMinutes, actionPrompt, sessionDuration, requiresConfirmation

UI: Active session timer (total elapsed) + notification card showing when next action is due + confirmation button ("Done ✓")

Behaviour: Sends a notification every intervalMinutes. User confirms the action was done. Tracks compliance rate over session.

Used for: 20-20-20 Rule (every 20 mins: look 20ft away for 20s)

Engine 3 — Prompt & Guided Task Engine

📝
GuidedPromptEngine
For step-by-step methodology guides
Parameters: steps[] (each with title, prompt, inputType), completionMessage

UI: Step progress indicator (pill dots at top) + current step title + instruction card + user input area (text / checkbox / free-write) + Next button

Behaviour: User moves through each step linearly. Final step shows summary of what they wrote. Session saved to history.

Used for: Feynman Technique, Cornell Note Method, SQ3R Method, Mind Mapping setup, Elaborative Interrogation

Engine 4 — Smart Task Sorter Engine

🗂
SmartTaskSorterEngine
For task prioritisation rules
Parameters: sortMode ("frog" | "135" | "pareto"), slots (e.g. {big:1, medium:3, small:5}), prompt

UI: Task input fields organised into rule-defined slots. "Frog" shows 1 highlighted task slot. "1-3-5" shows three sections. "80/20" shows a two-column split (vital few vs trivial many).

Behaviour: User fills the slots. App saves these as today's task structure. Feeds directly into the Todo screen with rule-tagged tasks. Completing all slots awards bonus points.

Used for: Eat the Frog, 1-3-5 Rule, 80/20 Rule (partial — for task selection)

Engine 5 — Spaced Repetition Scheduler Engine

🧠
SpacedRepetitionEngine
For memory and review scheduling
Parameters: intervals[] (e.g. [1,3,7,14,30] days), topic, notifyBeforeMinutes

UI: Topic input + review schedule display (a visual timeline showing Day 1, Day 3, Day 7... dots) + "Mark as Reviewed" button for current session + upcoming review notifications

Behaviour: User enters a topic they just learned. App schedules notifications at the spaced intervals (1 day, 3 days, 7 days, 14 days, 30 days). Each review check-in earns points. SM-2 algorithm optional in v2.

Used for: Spaced Repetition (SRS), 1-4-7 Rule (simplified 3-step variant: 1/4/7 days)

Engine 6 — Awareness / Reflection Engine

🪞
AwarenessReflectionEngine
For habit awareness and deadline setting
Parameters: reflectionPrompts[], deadlineMode (boolean), countdownTarget

UI: Reflection cards the user swipes through + optional deadline input + commitment button ("I commit to this deadline")

Behaviour: Guides user to set self-imposed deadlines (Parkinson's Law) or take a 5-second action (5-Second Rule). For 5-Second Rule: shows a 5-4-3-2-1 countdown after user taps "I need to do something" — they must act before it hits 0.

Used for: Parkinson's Law, 5-Second Rule

Engine 7 — Free-Write / Recall Engine

✍️
FreeWriteRecallEngine
For memory retrieval and note-taking techniques
Parameters: mode ("blurt" | "recall" | "chunk"), topicPrompt, timerMode (optional), revealAfterSeconds

UI: Topic input → blank text area with "Write everything you remember" prompt → optional timer → Reveal button (shows what was in source material vs what they wrote)

Behaviour: For Active Recall / Blurting: user writes freely, then compares. For Chunking: user breaks a topic into sub-sections with bullet points. Session saved to history. Can share/export as text.

Used for: Active Recall, Blurting Method, Chunking, Interleaving (mode variant)

💡
Engineering note for developers: Define a RuleConfig TypeScript interface. Each rule is a JSON config object — no rule-specific React components needed. The engine components are generic and accept config: RuleConfig as a prop. New rules can be added by adding a config object — zero new UI code required.
// RuleConfig TypeScript interface
interface RuleConfig {
  id: string                   // e.g. "pomodoro"
  name: string                 // Display name
  category: CategoryId         // "learning" | "focus" | "productivity" | "study"
  engine: EngineType           // "countdown" | "interval" | "guided" | "sorter" | "spaced" | "awareness" | "freewrite"
  description: string          // Short description (2–3 sentences)
  whyItWorks: string           // The science behind it
  bestFor: string[]            // e.g. ["exam prep", "deep work"]
  engineConfig: CountdownConfig | IntervalConfig | GuidedConfig | ...  // Engine-specific params
  pointsPerCompletion: number  // Default 15–20
  discoveryBonus: number       // 25 points first time
}

15 / Design
Light / Dark Theme
🌓
From the screenshots, the app is shown in dark mode (pure black background, white cards). Light mode inverts this — white background, off-white cards, dark text. The toggle is top-right on the home screen. Theme state persists via AsyncStorage and can optionally follow system preference.

Theme Toggle Logic

// theme/ThemeContext.tsx
type ThemeMode = 'light' | 'dark' | 'system'

const ThemeContext = createContext({ mode: 'dark', toggleTheme: () => {} })

export const ThemeProvider = ({ children }) => {
  const [mode, setMode] = useAsyncState('theme_mode', 'dark')  // persisted
  const systemMode = useColorScheme()  // 'light' | 'dark'
  const resolvedMode = mode === 'system' ? systemMode : mode

  return (
    <ThemeContext.Provider value={{ mode: resolvedMode, toggleTheme: () =>
      setMode(prev => prev === 'dark' ? 'light' : 'dark')
    }}>
      {children}
    </ThemeContext.Provider>
  )
}
TypeScript / React Native

Animation

Theme switch uses a fade transition (opacity: 1→0→1 over 200ms) or a radial reveal from the toggle button position (more impressive but optional for v1). All color values use the ThemeContext hook — no hardcoded colors anywhere in the component tree.
